# 國小一般排課系統 MVP 規格說明書

本系統是專為**國小一般排課情境**量身打造的排課系統 MVP。系統不分年級，全時段開放排課，並實作了高效率的「點選連點排課 (Click-to-Place)」、「教師/教室/班級硬性衝突偵測」以及「自動智慧切換教室」功能。

*   **快速入門**：關於詳細的安裝與啟動步驟（從安裝 Python 與 `uv` 開始），請參閱 [安裝與啟動指引 (INSTALL.md)](./INSTALL.md)。

## 📅 1. 時段結構與放學規則

*   **作息表**：每天共有 8 節課 + 中間 1 節午休。
    *   **上午**：第 1、2、3、4、5 節。
    *   **中午**：午休時段（無標記具體時間範圍，僅顯示「午休」字樣）。
    *   **下午**：第 6、7、8 節。
*   **放學規則**：取消所有年級的下午放學遮罩與限制，全天（週一至週五，第 1 至 8 節）均正常開放排課，週三下午亦同。

## 🏫 2. 智慧教室與自動切換機制

1.  **教室整併**：系統中移除了個別班級專屬普通教室的區分，統一整併為單一的「**班級教室**」(類型為 `普通`)。
2.  **智慧教室自動切換**：
    *   當管理員在左側**點選**或**拖曳**某門課程時，系統會根據課程設定的所需教室類型 (`classroom_type`)，自動變更右側的「安排授課教室」下拉選單：
        *   點選「國語 / 數學」➔ 自動切換為「**班級教室**」。
        *   點選「資訊」➔ 自動切換為「**電腦教室**」。
        *   點選「體育」➔ 自動切換為「**操場**」。
        *   點選「英文」➔ 自動切換為「**英語專用教室**」。
    *   管理員全程無需手動微調教室選單，可直接在課表點選排定多節課，系統會自動帶入正確的授課教室。

## ⚙️ 3. 核心衝突偵測引擎 (scheduler.py)

在排定（`POST`）或調整（`PUT`）課程時，系統會自動驗證以下 4 大硬限制，若違反則會阻擋並回傳衝突訊息：

1.  **班級衝突**：同一個班級在相同時間段 `(weekday, period)` 只能排定一節課。
2.  **教師衝突**：同一個教師在相同時間段只能在一處授課。
3.  **教室衝突**：同一個教室在相同時間段只能容納一門課程。
4.  **教師不排課時段**：避開教師設定的無效時間段（儲存於教師屬性中，例如 `["1-1"]` 代表週一第一節陳老師不可上課）。
5.  **教室類型不符**：課程指定的所需教室類型（如 `自然`）必須與所選教室類型一致。

*(註：支援單雙週排課規則，同一個時段若排入單週 (ODD) 與雙週 (EVEN) 課程，因週次錯開，將不被判定為衝突。但每週 (EVERY) 與單雙週重疊則視為衝突。)*

## 🖱️ 4. 前端交互功能與 Bug 修復

1.  **連點多節排課 (Click-to-Place)**：
    *   點擊左側待排池中的課程卡片，卡片會呈現青色高亮選取狀態。
    *   滑鼠直接點擊右側週曆網格中的空格，即可連續排課，不中斷選取狀態。
2.  **取消排課按鈕失效修復**：
    *   **Bug 描述**：由於已排課程卡片設定了 HTML5 的 `draggable="true"` 屬性，在大部分瀏覽器中，拖曳機制會吞噬內部的點擊事件，導致「x」取消排課按鈕點擊無效。
    *   **修復原理**：我們在 [app.js](./static/app.js) 中將取消排課事件從 `click` 改為監聽 **`mousedown`**，在拖曳判定生效前搶先觸發，順利解決此干擾。

## 📂 5. 專案架構與檔案路徑

```
/web/STC/
├── .venv/                  # Python 虛擬環境 (由 uv 管理)
├── pyproject.toml          # uv 依賴配置文件
├── database.py             # SQLite 連線與 Session 管理 (sqlite:///./scheduler.db)
├── models.py               # SQLAlchemy 資料庫 Model 定義
├── schemas.py              # Pydantic 資料驗證 Schema
├── scheduler.py            # 核心衝突偵測邏輯
├── main.py                 # FastAPI 路由進入點與範例資料初始化
├── static/                 # 前端網頁靜態資源
│   ├── index.html          # 主頁面
│   ├── style.css           # 毛玻璃極光風格樣式
│   └── app.js              # 前端事件與排課邏輯
└── tests/
    └── test_scheduler.py   # pytest 衝突檢測單元測試
```

    ```bash
    python -m uv run python -m pytest
    ```
